      # 以下推荐和 Acribus 配合使用

      # 资源和视图的自动注册

      1. 在项目 src 目录新建文件 requireAll.js ,并写入如下代码
      import acribus from 'acribus'
const resources = require.context('./', true, /\.res\.js$/)
const views = require.context('./', true, /\.view\.js$/)
const getNameFromFile = file => {
  const namePattern = /\S*\/([^/.]+)\.(?:res|view)\.js/
  const match = namePattern.exec(file)
  return match && match[1]
}
const requireAll = (context, handler) => {
  context.keys().forEach(key => {
    const item = context(key).default
    const name = getNameFromFile(key)
    handler(name, item)
  })
}

export const init = () => {
  requireAll(resources, (name, item) => {
    acribus.resource(name, item)
  })
  requireAll(views, (name, item) => {
    acribus.register(item)
  })
}
      1. 在 acribus.bootstrap 的回调函数初始化

      通常 acribus.bootstrap 写在 app 入口文件 src/main.js，在 main.js 写入下面代码

      // 引入文件
import {init as requireAllInit} from './requireAll'

acribus.bootstrap(() => {
  /* some your code */
  requireAllInit()
})
      1. Done
      • src 下所有文件被命名为 xxx.res.js 会被自定注册为资源
      • src 下所有文件被命名为 xxx.view.js 会被自定注册为视图

      TIP

      resource 声明文件放在 resource 文件夹下不是必须的，可以根据需要把相关的 resource 及 view 文件组织在一块

      # axure/接口API文档 转 Acribus 配置

      1. 全局安装 acr-cli
      sudo npm i acr-cli -g

      如果安装的时候报错，尝试执行下面的命令:

      sudo env PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=true npm i acr-cli -g
      1. 在项目根目录下添加 acribus.json:
      {
  "projectId" : 5000, // 后端 gitlab 仓库 Project ID
  "apiDocPath": "docs/api/" // api 文档在代码中的存放路径
}
      1. 在项目根目录下执行 acr gene 按提示输入即可
      在 GitHub 上编辑此页
