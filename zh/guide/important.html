# 教程

# Acribus 设计原则

  1. 所有行为可被重载
  2. 所有执行时机可控
  3. 所有执行进度可被感知

# 所有行为可被重载

Acribus 中默认行为的修改放在 options/interceptors 中。

包含下面几个行为:

  • 默认导航行为的重置: beforeEnter 可以在这里决定是否终止导航

  • 拉取资源详情时：one 可以在这里修改请求参数，或是终止请求

  • 拉取资源列表时：search 可以在这里修改请求参数，或是终止请求

  • 创建一条资源记录时：create 可以在这里修改请求参数，或是终止请求

  • 更新一条资源记录时：update 可以在这里修改请求参数，或是终止请求

# 所有执行进度可被感知

在 Acribus 中，store 是视图的唯一映射，这使得对视图变化（store 的改变）的观测变得很容易： 只需要订阅 store 的 mutation 或者 action 即可。注意这种订阅是全局的，在整个应用中只能被执行一次。如果放在组件实例化中，要在组件销毁的时候取消订阅。

Acribus options/subscribe 很适合放置这种订阅。

Acribus 中可以订阅的 mutation 和 action 见下面的 Refs

# 所有执行时机可控

所有的执行时机可控，是目标。 Acribus 每个行为都有默认的执行时机，比如 search 发生在用户点击 search 按钮后，更新发生在用户输入框值改变（表格内编辑模式）。

建议这样对执行时机进行控制：直接终止默认行为的执行（所有行为可被重载：interceptors） 然后订阅合适的时机，执行自定义行为

# Refs

  • 可以订阅的 mutation：

每个资源的每个 RESOURCE_ACTION_TYPE 都有下面三个 mutation：

RESOURCE_ACTION_TYPE = {
  LIST: 'list',
  ONE: 'one',
  ADD: 'add',
  UPDATE: 'update',
  DEL: 'delete'
}
  1. ${resourceName}/${RESOURCE_ACTION_TYPE}Start: 开始发起请求
  2. ${resourceName}/${RESOURCE_ACTION_TYPE}Success: 请求成功返回
  3. ${resourceName}/${RESOURCE_ACTION_TYPE}Error: 请求失败

eg:

person/listStart 会在 person 资源开始发起 search 时触发

  • 可以订阅的 action 参见关于 store 的介绍
